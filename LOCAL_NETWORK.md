# Режимы игры

В приложении три способа играть:

1. **🚀 Создать игру** — онлайн/LAN: участники подключаются со своих устройств,
   синхронизация в реальном времени. Для игры без интернета — см. ниже
   «Игра по локальной сети».
2. **📺 Один экран (трансляция)** — всё на одном компьютере, см. раздел
   «Режим одного экрана» ниже. Ничего поднимать не нужно.
3. **✎ Редактор** — создание режимов и вопросов.

---

# Режим одного экрана (трансляция)

Когда ведущий показывает игру на большом мониторе/проекторе со **своего**
компьютера (HDMI или демонстрация экрана в Zoom/Meet) и **сам** кликает по
клеткам. Участникам подключаться не нужно.

- На главной нажми **«📺 Один экран (трансляция)»** (или открой `/solo`).
- Выбери команды и режим → играй.
- При клике по клетке открывается вопрос, **ответ скрыт**. Ответ появляется
  только после кнопки **«👁️ Показать ответ»** — поэтому можно спокойно
  транслировать этот экран, спойлера не будет.
- Интернет и сервер не нужны: всё происходит в браузере. Прогресс сохраняется,
  переживает перезагрузку страницы.

> Совет: `F11` — полноэкранный режим браузера для трансляции.

# Игра по локальной сети БЕЗ интернета

Этот режим позволяет провести игру, когда интернета нет вообще. Вместо облака
Firebase синхронизацию берёт на себя локальный сервер на твоём MacBook, а
участники подключаются к нему по Wi-Fi роутера.

---

## Что понадобится

- MacBook (на нём крутится сервер и табло ведущего).
- Wi-Fi роутер. **Интернет ему не нужен** — он работает как обычная локальная сеть.
- Телефоны/планшеты/ноутбуки участников — в той же Wi-Fi сети роутера.

> Лучше всего подключить MacBook к роутеру **кабелем** (Ethernet) — максимально
> стабильно. Если кабеля нет — по Wi-Fi к тому же роутеру.

---

## Подготовка ДОМА (пока есть интернет) — один раз

```bash
npm install          # поставить зависимости (нужно один раз)
```

Если режим игры создавался в **редакторе** (а не лежит в `public/data`), выгрузи
его в локальный файл, пока есть интернет:

```bash
npm run export:modes
```

> Готовые режимы (divewing26, mms, outreach3 и т.п.) уже встроены в приложение и
> работают офлайн — для них этот шаг не нужен. `export:modes` нужен только для
> режимов, собранных в редакторе.

Собери приложение в локальном режиме:

```bash
npm run build:local
```

---

## На мероприятии (без интернета)

1. Включи роутер. Подключи к нему MacBook (кабелем или по Wi-Fi).
2. Запусти сервер:

   ```bash
   npm run local
   ```

   (если уже собирал `build:local` и не менял код — можно быстрее: `npm run local:serve`)

3. В терминале появятся адреса вроде:

   ```
   📡 Адреса для участников в этой Wi-Fi сети:
      ➜  http://192.168.1.50:3000
   ```

   Возьми тот, что начинается на `192.168.…` (адрес MacBook в сети роутера).

4. **Ведущий** открывает этот адрес в браузере MacBook, создаёт комнату — получает код.
5. **Участники** в браузере телефона открывают тот же `http://192.168.1.50:3000`,
   вводят код комнаты и выбирают команду.
6. Табло для проектора: `http://192.168.1.50:3000/room/КОД/display`.

Остановить сервер — `Ctrl+C`.

---

## Сколько человек выдержит

- MacBook-сервер — не узкое место (сотни подключений).
- Реальный предел — роутер: обычно комфортно 30–50 устройств, что перекрывает
  почти любую командную игру.

---

## Как это устроено (для разработчика)

- Выбор бэкенда — переменной сборки `VITE_BACKEND`:
  - не задана → **Firebase** (онлайн, деплой на Vercel) — поведение по умолчанию;
  - `=local` → локальный сервер по WebSocket.
- Весь код ходит в базу через [src/lib/rtdb.ts](src/lib/rtdb.ts), а не напрямую в
  `firebase/database`. Этот фасад подставляет нужный бэкенд.
- Локальный клиент: [src/lib/localBackend.ts](src/lib/localBackend.ts) —
  drop-in замена `ref/set/get/onValue/remove/update/runTransaction/serverTimestamp`
  с авто-реконнектом.
- Сервер: [local-server.js](local-server.js) — раздаёт `dist/` и держит дерево
  данных в памяти, сохраняя его в `local-data/db.json` (переживает перезапуск).
- Состояние игры (`local-data/`) в git не коммитится.

Онлайн-режим Firebase при этом полностью сохранён — обычный `npm run build` и
деплой на Vercel работают как раньше.
